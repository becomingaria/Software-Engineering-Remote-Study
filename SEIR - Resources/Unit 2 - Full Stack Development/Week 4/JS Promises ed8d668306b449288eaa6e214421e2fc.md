# What is a promise?

"A *Promise* is an object representing the eventual completion or failure of an *asynchronous operation*." - [source](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Using_promises)

## What is it?

Once a promise has been called, it will start in a `pending` state. This means that the calling function continues executing, while the promise is pending until it resolves, giving the calling function whatever data was being requested.

The created promise will eventually end in a `resolved` state, or in a `rejected` state, calling the respective callback functions (passed to then and catch) upon finishing.

## The syntax of a Promise

```jsx
const promise1 = doSomething().then(successCallback, failureCallback);
```

```jsx
	doSomething()
    .then(result => doSomethingElse(result))
    .then(newResult => doThirdThing(newResult))
    .then(finalResult => {
    console.log(`Got the final result: ${finalResult}`);
    })
    .catch(failureCallback);

```

is the same as:

```jsx
	doSomething()
    .then(function(result) {
    return doSomethingElse(result);
    })
    .then(function(newResult) {
    return doThirdThing(newResult);
    })
    .then(function(finalResult) {
    console.log('Got the final result: ' + finalResult);
    })
    .catch(failureCallback);
```

## How does it work?

1. `doSomething` is a function that returns a promise
2. The promise will invoke an anonymous function provided to `resolve` the promise. That value is passed to the promises built in `.then` method as an argument.  
3. The `.then` method will invoke the callback with access to the `fulfilled` promise’s value. The callback can also return a value (such as the `doSomethingElse` function). The purpose of the callback is provide scoping for promises resolver and to access the promises data. 
4. With the first `.then` resolved, the resulting data from `doSoemthingElse` is provided to the next anonymous function as an argument `newResult`
5. The result of `doThirdThing`  is finally passed through our 3rd `.then` invocation with `doThirdThing`'s return valued provided as `finalResult` parameter. 
6. Any errors that occur in any of the callback functions is ‘caught’ in the `.catch()` promise method. 

This process of returning a value from a .then() is called `promise chaining` because each .then() itself returns a promise. 

## Why use promises?

Imagine that you need to work with code that takes time, either because you are accessing data on a remote server, or you want to delay some process.

You can define a function that instead of returning the data directly, returns a promise - a promisified function returns a promise object. JS provides the `promise` two helper functions `resolve` and `reject` that can be called in your function depending on a given condition.

**If some success condition is met:**
We can call the `resolve()` to return the data we want,

**If there is some error or our code does not return in a desired state:**
we can call `reject()` and pass along relevant information (i.e. the error data).

## In Node - file access

```jsx

const fs = require('fs')

const getFile = (fileName) => {
  return new Promise((resolve, reject) => {
    fs.readFile(fileName, (err, data) => {
      if (err) {
        reject(err)  // calling `reject` will cause the promise to fail with or without the error passed as an argument
        // and we don't want to go any further
      }
      resolve(data)
    })
  })
}
```

## In the browser

```jsx
fetch(endpoint, options).then(response => {
  console.log(response)
  // when you make a request, the response from the 
  // server is returned as our 'response value'

  return response.json()
  
// vocab: serialization -> converting one datatype to another (json to JS)
  // takes the response body's (data)

.then(user => {
    console.log(user.results[0])
  })

    console.log(err.message)

  })
```

## How do we write our own `promisified` function?

```jsx
let done = true

const isItDoneYet = () => new Promise((resolve, reject) {
  if (done) {
    const workDone = 'Here is the thing I built'
    setTimeout(()=> resolve(workDone), 1000)
  } else {
    const why = 'Still working on something else'
		setTimeout(()=> reject(why), 1000)
  }
})

const checkIfItsDone = () => {
  isItDoneYet()
    .then(ok => {
      console.log(ok)
    })
    .catch(err => {
      console.error(err)
    })
}

checkIfItsDone()
```

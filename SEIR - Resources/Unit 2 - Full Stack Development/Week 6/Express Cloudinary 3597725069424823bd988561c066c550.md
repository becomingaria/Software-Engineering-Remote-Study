## Installation of Multer

To install the Multer library run `npm install multer` and import the package into your express `postsController` application file. This middleware could be configured globally, but for demonstration purposes we will implement the functionality at the controller level.

## Setup for  `Multer` middleware

Multer is processing our multipart/form-data, which is primarily used for uploading files.

Multer is designed to capture the incoming requests and manipulate the `request` object property if a file or files object to the request. The `req.body` object contains the values of the text fields of the form while file datat will be available in our controllers in a `req.files` property. The file or files object contains the files uploaded via the form, including relevant file metadata (type, size, path, etc).

```jsx
// /routers/posts.js

const multer = require("multer");
...
const upload = multer({ dest: "uploads/test" });
```

When configuring your multer middleware you can experiment with its functionality in the context of your app’s router. You can configure multer to be a global or controller specific middleware; this decision would be determined by your application’s architecture and user needs.  

Multer will accept an options object, the most basic of which is the `dest` property; the `dest` tells multer where to upload the files to your server. 

In the case you omit the options object, the files will be kept in memory as a `buffer stream` type and never written to disk; this approach will be utilized later when we integrate our Cloudinary SDK to stream data to a cloud storage server. 

## Demonstrating`Multer` methods

When building your form you will want to match the name of the `type=“file”` input's `name`  attribute to the string you pass `upload` method.

### `single` - **singleFileUpload**

```jsx
router.post(
  "/:id/photos/single",
  upload.single("imageUpload"),
  function (req, res, next) {
    console.log(req.file);
    console.log(req.body);
    // req.file is the `imageUpload` file this string should match the name you specified on the upload input
    // req.body will hold the text fields, if there were any our redirect after upload
    res.redirect("/");
  }
);
```

### `array` - a ‘**multi’ file upload middleware**

```jsx
router.post(
  "/:id/photos/multi",
  upload.array("inputUpload", 6),
  function (req, res, next) {
    // req.files is array of `photos` files
    // req.body will contain the text fields, if there were any our redirect after upload
    res.redirect("/");
  }
);

```

## Updating show.ejs template

The form will only complete this request after a submit event is fired; any non-file form data provided with the form request will be passed to the request body. 

This information will be accessible from the request object via the `req.body` prop, and the file information from the `req.file` (single)/ `req.files` (multi) props.  While `multipart/form-data` may be new, but allows us to configure our request/data-type and is *required* if you want `multer` to work properly.

```html
<form method="POST" 
	autocomplete="off" 
	action="/posts/<%=post._id%>/photos/single" 
	enctype="multipart/form-data"
>
    <label>
        <h3>Image Description</h3>
        <input type="file" name="imageUpload" required />
    </label>
    <label>
        <h3>Image Description</h3>
        <input type="text" name="description"/>
    </label>
    <label>
        <h3>Image Alt Text</h3>
        <input type="text" name="alt"/>
    </label>
    <br />
    <input type="submit" value="Create Post">
</form>

```

Reminder: The form **must** have an attribute `enctype="multipart/form-data"` or the multer image parsing functionality will not work. 

Form data encoding provide the server with information about the file type and content sent with the upload form’s POST [request.](http://request.You) You can learn more about how multi-part form data and file uploads are handled by the browser [here](https://developer.mozilla.org/en-US/docs/Web/API/FormData/Using_FormData_Objects).

## Setting up Cloudinary

### About Cloudinary

"Cloudinary is an end-to-end image and video-management solution for websites and mobile apps, covering everything from image and video uploads, storage, manipulations, optimizations to delivery."

The Cloudinary service offers an API service, allowing easy access with fetch within the browser.  

However, Cloudinary’s NodeJS SDK provides streaming/uploading tools we can integrate into our express application. This server side approach is what we will be demoing this morning.

### Installation of Cloudinary


We need a route to intercept the uploaded file from our form on the post show page. In order to link your form’s POST request, create a route which matches the `action` path for your upload form. Between the configured route string, we will keep the multer middleware in place so the `uploadPhoto` function has access to the uploaded files.  

### Cloudinary Upload Controller

```jsx
// routes/posts.js

async function uploadPhoto(req, res, next) {
    try {
      let result = await streamUpload(req);
      console.log(req.body);
      console.log(result);
			// additional database updates could occur after the content is uploaded. 
      res.redirect(`/posts/${req.params.id}`);
    } catch (err) {
        console.log(err)
	      next(err)
    }
}
```

The `uploadPhoto` controller is responsible for the following functionality:

- call a promisified function that will handle the asynchronous streaming to the Cloudinary Servers
- The returned `result` from `streamUpload` is then available for manipulation.
- Possible extensions of this base upload functionality:
    - Once the `result` (response from cloudinary) is complete - now it is time to update your database documents with the created uploaded image url. Find the associated post and update our DB with information about the new image url from the Cloudinary response data; the `alt` and `description` content would be accessed through the request body.
    - This update would require appending a new child sub-document to the current Post's `images` array and saving the parent document.
    - After the update occurs a redirect back to the show page would result in the images state being available to render.

### Cloudinary + `streamUpload` Promise

```jsx
function streamUpload (req) {
    return new Promise((resolve, reject) => {
      let stream = cloudinary.uploader.upload_stream((error, result) => {
        if (result) {
          console.log(result);
          resolve(result);
        } else {
          reject(error);
        }
      });
      streamifier.createReadStream(req.file.buffer).pipe(stream);
    });
};
```

The uploading function `streamUpload` returns a promise and is responsible for transferring the buffer data (`req.file.buffer`) and send the file meta data to Cloudinary server. 

The `upload_stream` callback will return an error that we pass with our promise’s `reject` function. If there is no error in uploading your file to Cloudinary an object with information about the uploaded file / status will be resolved once the promise is fulfilled.

### (Optional) Configuring your file upload name and location on Cloudinary

The upload stream can be configured with an optional argument `{}` that allows for various cloudinary configurations upon upload. This is a great use case for a config sub module helper function. 

```jsx
// inside `streamUpload` promise executor function 
// optional way create a unique-custom filename for cloudinary 

			// access the file name from the req.file object
      const filename = req.file.originalname.split('.')[0]
			// use the alt text to name the file
      const altText = req.body.alt.replace(/ /g,'-').toLowerCase()
			// create the custom string
      const identifier = `${filename}-${altText}`
      
			// create a config object - this will be passed as argument to upload_stream before the callback
      const uploadConfig = {
        public_id: identifier,
        folder: '/demo/assets'
      }
```
